Git中对于删除首次提交的看法

2021-06-06
1分钟阅读时长
Git

update-ref指令：

git update-ref -d HEAD

它将删除命名引用。HEAD，所以它会重新设置(轻轻地，不会丢失工作当前分支的所有提交

如果要将第一个提交与第二个提交合并，则可以使用rebase指令：

git rebase -i --root

最后一种方法可以是创建一个孤立分支，一个具有相同内容但没有任何提交历史的分支，并在其上提交新内容：

git checkout --orphan <new-branch-name>

在第一次提交之前没有任何内容，因为每个提交都是指父提交。这使得第一次提交是特殊的(孤儿提交)，因此无法引用以前的"状态"。

因此，如果想要修复提交，可以简单地git commit --amend，这将修改提交而不创建另一个提交

如果只想重新开始，请删除.git文件夹并重新使用git init初始化本地仓库
